---
title: 'Parsing PL/pgSQL'
description: 'Use pg_query_parse_plpgsql() to parse PL/pgSQL function bodies into a JSON representation.'
---

`pg_query_parse_plpgsql()` parses one or more PL/pgSQL function definitions and returns a JSON array describing each function's body. This is useful for analyzing stored procedures — for example, to extract the SQL expressions they contain, detect patterns, or lint PL/pgSQL code.

## Function signature

```c
PgQueryPlpgsqlParseResult pg_query_parse_plpgsql(const char* input);
```

The `input` parameter should be a complete `CREATE FUNCTION` or `CREATE OR REPLACE FUNCTION` statement that uses `LANGUAGE plpgsql`.

### Result struct

```c
typedef struct {
  char* plpgsql_funcs;  // JSON array of parsed function bodies, or NULL on error
  PgQueryError* error;  // NULL on success, populated on parse error
} PgQueryPlpgsqlParseResult;
```

`plpgsql_funcs` is a JSON array with one element per function found in the input. Each element is a `PLpgSQL_function` object whose structure mirrors PostgreSQL's internal PL/pgSQL representation.

## Example

```c
#include <pg_query.h>
#include <stdio.h>
#include <stdlib.h>

int main() {
  PgQueryPlpgsqlParseResult result;

  result = pg_query_parse_plpgsql(" \
  CREATE OR REPLACE FUNCTION cs_fmt_browser_version(v_name varchar, \
                                                  v_version varchar) \
RETURNS varchar AS $$ \
BEGIN \
    IF v_version IS NULL THEN \
        RETURN v_name; \
    END IF; \
    RETURN v_name || '/' || v_version; \
END; \
$$ LANGUAGE plpgsql;");

  if (result.error) {
    printf("error: %s at %d\n", result.error->message, result.error->cursorpos);
  } else {
    printf("%s\n", result.plpgsql_funcs);
  }

  pg_query_free_plpgsql_parse_result(result);

  // Optional: frees the top-level memory context
  pg_query_exit();

  return 0;
}
```

## Sample output

```json
[
  {
    "PLpgSQL_function": {
      "datums": [
        {
          "PLpgSQL_var": {
            "refname": "found",
            "datatype": {
              "PLpgSQL_type": { "typname": "UNKNOWN" }
            }
          }
        }
      ],
      "action": {
        "PLpgSQL_stmt_block": {
          "lineno": 1,
          "body": [
            {
              "PLpgSQL_stmt_if": {
                "lineno": 1,
                "cond": {
                  "PLpgSQL_expr": { "query": "SELECT v_version IS NULL" }
                },
                "then_body": [
                  {
                    "PLpgSQL_stmt_return": {
                      "lineno": 1,
                      "expr": {
                        "PLpgSQL_expr": { "query": "SELECT v_name" }
                      }
                    }
                  }
                ]
              }
            },
            {
              "PLpgSQL_stmt_return": {
                "lineno": 1,
                "expr": {
                  "PLpgSQL_expr": {
                    "query": "SELECT v_name || '/' || v_version"
                  }
                }
              }
            }
          ]
        }
      }
    }
  }
]
```

Notice that SQL expressions inside PL/pgSQL (conditions, return values, assignments) are captured as `PLpgSQL_expr` nodes with a `query` field containing the SQL text prefixed with `SELECT`. You can pass these `query` strings to `pg_query_parse()` for further analysis.

## Error handling

If the input is not valid PL/pgSQL, `result.error` is non-NULL:

```c
if (result.error) {
  printf("error: %s at %d\n",
         result.error->message,
         result.error->cursorpos);
}
```

<Note>
Always call `pg_query_free_plpgsql_parse_result(result)` after you are done with the result. This releases the memory allocated for `plpgsql_funcs` and `error`.
</Note>
