---
title: "Language wrappers"
description: "Higher-level libraries built on libpg_query for Ruby, Go, Rust, JavaScript, Python, OCaml, and more."
---

libpg_query is primarily a base C library. Most applications use a higher-level language wrapper instead of calling the C API directly.

<Note>
  Each wrapper has its own API that may differ significantly from the C API documented in this reference. Consult the wrapper's own documentation for usage instructions.
</Note>

---

## Wrappers by language

  <Card

  <Card

## Projects built on libpg_query

The following products, tools, and libraries use libpg_query (or a language wrapper) in production:

- [pganalyze](https://pganalyze.com/) — PostgreSQL performance monitoring
- [hsql](https://github.com/JackDanger/hsql) — SQL query manipulation tool
- [sqlint](https://github.com/purcell/sqlint) — SQL linting tool
- [pghero](https://github.com/ankane/pghero) — PostgreSQL performance dashboard
- [dexter](https://github.com/ankane/dexter) — automatic index advisor
- [pgscope](https://github.com/gjalves/pgscope) — query dependency visualization
- [pg_materialize](https://github.com/aanari/pg-materialize) — materialized view manager
- [DuckDB](https://github.com/cwida/duckdb) — uses libpg_query's parser as an embedded component
- [pgspot](https://github.com/timescale/pgspot) — PostgreSQL extension security analysis
- [Postgres Language Server](https://github.com/supabase/postgres_lsp) — LSP-based SQL editor tooling
