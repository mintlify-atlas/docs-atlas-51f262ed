## Error type

### `PgQueryError`

Returned via a pointer field named `error` on every result struct. When `error` is `NULL` the call succeeded.

```c
typedef struct {
    char* message;
    char* funcname;
    char* filename;
    int   lineno;
    int   cursorpos;
    char* context;
} PgQueryError;
```

<ResponseField name="message" type="char*" required>
  Human-readable exception message.
</ResponseField>

<ResponseField name="funcname" type="char*" required>
  Name of the PostgreSQL source function that raised the exception (e.g. `SearchSysCache`).
</ResponseField>

<ResponseField name="filename" type="char*" required>
  Name of the PostgreSQL source file where the exception originated (e.g. `parse.l`).
</ResponseField>

<ResponseField name="lineno" type="int" required>
  Line number in the PostgreSQL source file.
</ResponseField>

<ResponseField name="cursorpos" type="int" required>
  Character offset in the input query string where the exception occurred.
</ResponseField>

<ResponseField name="context" type="char*">
  Optional additional context string. May be `NULL`.
</ResponseField>

## Binary buffer

### `PgQueryProtobuf`

A length-prefixed byte buffer used to hold serialized Protobuf messages.

```c
typedef struct {
    size_t len;
    char*  data;
} PgQueryProtobuf;
```

<ResponseField name="len" type="size_t" required>
  Length of the serialized Protobuf data in bytes.
</ResponseField>

<ResponseField name="data" type="char*" required>
  Raw serialized Protobuf bytes. Pass to the appropriate `protobuf-c` unpack function.
</ResponseField>

## Deparse option types

### `PostgresDeparseOpts`

Controls output formatting for [`pg_query_deparse_protobuf_opts`](/reference/api#pg_query_deparse_protobuf_opts). Defined in `postgres_deparse.h`.

```c
typedef struct PostgresDeparseOpts {
    PostgresDeparseComment** comments;
    size_t                   comment_count;
    bool   pretty_print;
    int    indent_size;
    int    max_line_length;
    bool   trailing_newline;
    bool   commas_start_of_line;
} PostgresDeparseOpts;
```

<ResponseField name="comments" type="PostgresDeparseComment**">
  Array of comments to re-insert into the deparsed output. Obtain via [`pg_query_deparse_comments_for_query`](/reference/api#pg_query_deparse_comments_for_query). May be `NULL`.
</ResponseField>

<ResponseField name="comment_count" type="size_t">
  Number of entries in the `comments` array.
</ResponseField>

<ResponseField name="pretty_print" type="bool">
  When `true`, emit human-readable multi-line output. Default: `false`.
</ResponseField>

<ResponseField name="indent_size" type="int">
  Number of spaces per indentation level when `pretty_print` is enabled. Default: `4`.
</ResponseField>

<ResponseField name="max_line_length" type="int">
  Maximum line length for wrapping certain lists of items when `pretty_print` is enabled. Default: `80`.
</ResponseField>

<ResponseField name="trailing_newline" type="bool">
  When `true`, append a trailing newline to the output. Default: `false`.
</ResponseField>

<ResponseField name="commas_start_of_line" type="bool">
  When `true`, place separating commas at the start of the next line rather than the end of the current line. Default: `false`.
</ResponseField>

### `PostgresDeparseComment`

Describes a single SQL comment extracted from a query, for re-insertion during deparsing. Defined in `postgres_deparse.h`.

```c
typedef struct PostgresDeparseComment {
    int   match_location;
    int   newlines_before_comment;
    int   newlines_after_comment;
    char* str;
} PostgresDeparseComment;
```

<ResponseField name="match_location" type="int" required>
  Insert this comment before the first parse-tree node whose `location` field is greater than or equal to this value.
</ResponseField>

<ResponseField name="newlines_before_comment" type="int" required>
  Number of newlines to insert before the comment. Non-zero when the original comment was separated from the preceding token by at least one newline.
</ResponseField>

<ResponseField name="newlines_after_comment" type="int" required>
  Number of newlines to insert after the comment. Non-zero when the original comment was separated from the following token by at least one newline.
</ResponseField>

<ResponseField name="str" type="char*" required>
  Full comment text, including the comment start/end tokens (e.g. `/* … */` or `-- …`) and any embedded newline characters.
</ResponseField>

## Enums and constants

### `PgQueryParseMode`

Controls what the parser treats the input as. Pass as the low-order bits of the `parser_options` integer using functions that end in `_opts`.

```c
typedef enum {
    PG_QUERY_PARSE_DEFAULT         = 0,
    PG_QUERY_PARSE_TYPE_NAME,
    PG_QUERY_PARSE_PLPGSQL_EXPR,
    PG_QUERY_PARSE_PLPGSQL_ASSIGN1,
    PG_QUERY_PARSE_PLPGSQL_ASSIGN2,
    PG_QUERY_PARSE_PLPGSQL_ASSIGN3
} PgQueryParseMode;
```

| Value | Description |
| ----- | ----------- |
| `PG_QUERY_PARSE_DEFAULT` | Standard SQL query parsing (the default). |
| `PG_QUERY_PARSE_TYPE_NAME` | Parse input as a type name. |
| `PG_QUERY_PARSE_PLPGSQL_EXPR` | Parse input as a PL/pgSQL expression. |
| `PG_QUERY_PARSE_PLPGSQL_ASSIGN1` | Parse input as a PL/pgSQL assignment (target variant 1). |
| `PG_QUERY_PARSE_PLPGSQL_ASSIGN2` | Parse input as a PL/pgSQL assignment (target variant 2). |
| `PG_QUERY_PARSE_PLPGSQL_ASSIGN3` | Parse input as a PL/pgSQL assignment (target variant 3). |

The parse mode occupies the lowest 4 bits of `parser_options` (`PG_QUERY_PARSE_MODE_BITS = 4`).

---

### Parser option constants

Bitwise-OR these flags into `parser_options` alongside a `PgQueryParseMode` value to change GUC settings that affect parsing.

```c
#define PG_QUERY_DISABLE_BACKSLASH_QUOTE              16
#define PG_QUERY_DISABLE_STANDARD_CONFORMING_STRINGS  32
#define PG_QUERY_DISABLE_ESCAPE_STRING_WARNING        64
```

| Constant | Value | Effect |
| -------- | ----- | ------ |
| `PG_QUERY_DISABLE_BACKSLASH_QUOTE` | `16` | Sets `backslash_quote = off`. The default is `safe_encoding`, which is effectively on. |
| `PG_QUERY_DISABLE_STANDARD_CONFORMING_STRINGS` | `32` | Sets `standard_conforming_strings = off`. The default is on. |
| `PG_QUERY_DISABLE_ESCAPE_STRING_WARNING` | `64` | Sets `escape_string_warning = off`. The default is on. |

**Example — combine parse mode and flags:**

```c
int opts = PG_QUERY_PARSE_TYPE_NAME
         | PG_QUERY_DISABLE_STANDARD_CONFORMING_STRINGS;
PgQueryParseResult result = pg_query_parse_opts("text", opts);
```
